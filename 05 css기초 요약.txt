html 요소를 웹문서에 넣어 뼈대를 만드는것
css 웹문서의 디자인 요소담당

css기초 (Cascading Style Sheets)

style sheets  한문서에서 반복해서 사용하는 스타일들을 모아놓은것

style= HTML문서에서 자주 사용하는 글꼴 색상 정렬 각 요소들의 배치 방법등 문서의 겉모습을 결정짓는 내용들
style 사용하는 이유
	1. 내용과 상관없이 디자인만 변경가능
	2. 다양한 기기에 맞게 탄력적으로 바뀌는 문서 제작 가능 (대상 기기에 맞게 css만 바꿔주면 됨)

style 형식
	p { text-align: center;}
 	선택자 {스타일속성 : 속성 값}
	

선택자selector : 스타일 규칙을 어디에 적용할지
{중괄호}	: 사이에 속성과 속성값써줌
 : 	: 속성 속성값구분
 ; 	: 속성들간 구분


스타일에도 주석달아서 설명해주면 좋음 /* */


stylesheets
	스타일 규칙들을 한군데 묶어 둔 것
	-내부 스타일 시트 : 웹문서에 사용할 스타일을 문서안에 정리한것
			<head> 
				<style>
					내부스타일 시트
				</style>
			</head> 
	-외부 스타일 시트 : 여러 웹문서에서 사용할 스타일을 별도의 파일로 저장해 따로 저장
			.css 파일 확장자 사용
			<head> 
				<ling href="외부 스타일 파일 경로" rel="stylesheet" type="text/css">
			</head>

	-인라인 스타일 : 스타일 적용할 대상 태그에 직접 표시
			<p style="color:blue;"> 아아아아아</p>

		우선순위   인라인> 내부> 외부     외부스타일시트 가장 추천


selector 선택자
	-전체 선택자 : 스타일을 모든 요소에 적용할때 사용
			* { 속성:속성값; 속성:속성값;}
	-태그 선택자 : 특정 태그가 쓰인 모든 요소들에 적용
			태그 { 속성:속성값; 속성:속성값;}	
		태그와 요소의 차이 : ex <p>라라라라라 </p>   p가 태그이고  그 안의 라라라라라가 요소

	-클래스 선택자 : 같은 태그라도 다른 스타일을 사용가능케 함. 
		         id 태그와의 차이점 : 클래스가 같으면 여러개 태그에 같은 스타일 적용가능
			.class이름{ 속성:속성값; 속성:속성값;}
		    	태그선택자와 클래스 선택자를 동시에 사용도 가능
			h2.class이름{ 속성:속성값; 속성:속성값;}
			h2태그의 class이름 부여된 요소들만 스타일 적용 클래스가같더라도
			다른 태그는 스타일 적용안됨.
	
	-id 선택자 : 웹문서 안의 id를 부여한 요소에만 스타일 적용
		   클래스 선택자와 다르게 문서안에서 한번만 사용가능 유일성
			#아이디명 { 속성:속성값; 속성:속성값;}

	-그룹 선택자 : 둘이상의 요소에 같은 스타일 적용할때
			h1,h2{ 속성:속성값; 속성:속성값;} 
			쉼표로 구분해줌

Cascading 
	위에서 아래로 흐르는 스타일 시트
	선택자에 적용된 많은 스타일 중에 우선순위를 부여하고 어떤것을 나타낼지 정함

	스타일의 우선순위
		
	중요도
		1순위 사용자 스타일 시트가 최우선 (사용자 개인이 저장한것)
		2순위 제작자 스타일 시트 ( 제작하면서 만든 스타일시트)
			!important 를 붙이면 다른 스타일보다 최우선 적용 
		3순위 브라우저 스타일 시트(각 브라우저마다 기본적으로 지정된 스타일)

	명시도 
			스타일의 적용 범위가 좁을수록 우선순위 높음
		1 인라인 스타일-- 태그안에 스타일 속성을 적용시키는것 최우선
		2 id스타일		#
		3 클래스 스타일		.		
		4 태그 스타일		그냥태그명
	
	소스에서의 순서		
		
		나중에 온 스타일이 먼저온 스타일 덮어씀  -- 밑에있는것이 위에있는것 덮어씀
	
	스타일 상속
	
		부모 요소에 있는 스타일 속성은 자식요소로 전달되는데 이것을 스타일 상속이라함
		모든 속성이 상속되는것은 아님
